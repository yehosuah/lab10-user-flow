MercadoCerca

Flujo para pedir supermercado a domicilio

Usuario meta: persona ocupada que quiere recibir productos frescos el mismo dia, sin confundirse entre tienda, carrito, pago y horario.

Inicio/fin Pantalla Decision Sistema

Flujo activo

Pedido completo

Desde abrir la app hasta confirmar una entrega programada.

Objetivo
Enviar una orden de supermercado sin friccion.
Exito
Orden confirmada con tienda, productos, pago y horario.
Riesgo UX
Abandono si costos o disponibilidad aparecen tarde.
Usuario App Sistema

Nodo seleccionado

Abrir app

El usuario entra con intencion clara: comprar comida para entrega.

Inicio Usuario

Detalle UX

Mostrar direccion guardada y acceso rapido a tiendas cercanas.

Entrega

Decisiones clave para el equipo

  1. El costo total aparece antes de pedir el pago.
  2. La app valida inventario antes de elegir horario.
  3. Los casos de producto agotado tienen reemplazo, eliminar o guardar.
  4. Los flujos extra reutilizan el mismo checkout para reducir aprendizaje.
